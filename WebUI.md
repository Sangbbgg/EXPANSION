# 🚀 프로젝트 EXPANSION: 마스터 자동화 가이드 (Final)

본 파일은 **EXPANSION** 시스템의 자동화 루프, 프로젝트 독립성, 자가 감사(Self-Audit) 및 UI 표준을 정의합니다. Gemini CLI는 모든 명령 수행 시 이 가이드를 최우선으로 참조해야 합니다.

---

### [Phase 0: 운영 원칙 및 트리거 고정]

* [x] 0-1. **[자동화 트리거]** 명령 수신 시 반드시 본 파일과 해당 프로젝트의 `Context/*.md`를 선행 분석한다. 작업 완료 후에는 즉시 체크(`[x]`) 및 `git push`를 수행한다.
* [ ] 0-2. **[격리 원칙]** 모든 신규 프로젝트는 `/projects/[project_name]` 하위 독립 디렉토리에 구축한다. 프로젝트 간 코드 참조는 금지하며, 필요 시 DB/API로 통신한다.
* [ ] 0-3. **[추적 프로세스]** 모든 프로젝트는 생성 즉시 `/Context/[project_name]_spec.md`를 생성하여 단계별 계획을 문서화하고, 구현 완료 시마다 AI가 스스로 체크한다.
* [ ] 0-4. **[Git 프로세스 엄수]** `Everything up-to-date` 방지를 위해 모든 변경 사항은 반드시 `git add .` -> `git commit` -> `git push` 단계를 거친다.
* [ ] 0-5. **[UI 단일화]** 공통 네비게이션은 `layout.tsx`에서만 관리하며, 개별 페이지나 프로젝트 폴더 내에서 사이드바를 중복 호출하지 않는다.

---

### [Phase 1: UI/UX 정밀 조정 (화면 최적화)]

* [x] 1-1. **토글형 사이드바**: 왼쪽 네비게이터의 폭을 최적화하고, 클릭 시 아이콘 모드와 텍스트 모드로 전환되는 부드러운 토글 기능을 구현한다.
* [x] 1-2. **채팅창 고도화**: 'AI Chat Interface'에 고정 높이와 `overflow-y-auto`를 적용하여 내부 스크롤을 생성한다.
* [x] 1-3. **정보 밀도 최적화**: 'AI Chat Interface'와 'Live Logs'의 폰트 크기를 `text-sm` 또는 `text-xs`로 줄여 가독성을 높인다.
* [x] 1-4. **중복 레이아웃 제거**: 현재 2중으로 나타나는 네비게이터 버그를 즉시 수정하여 단일 레이아웃으로 통합한다.

---

### [Phase 2: 지시 수신 및 독립 프로젝트 구축]

* [ ] 2-1. 웹 UI 지시 수신 시 `/projects/[project_name]` 폴더와 그 내부의 `Context/` 폴더를 즉시 생성한다.
* [ ] 2-2. 요구사항을 분석하여 `Context/[project_name]_spec.md`에 기능 리스트를 작성하고 첫 번째 Task부터 구현을 시작한다.

---

### [Phase 3: 자가 구현 및 자가 치유(Self-Healing)]

* [ ] 3-1. 정의된 `_spec.md` 순서에 따라 코딩하며, 완료 시마다 AI가 문서를 직접 수정하여 진행률을 업데이트한다.
* [ ] 3-2. `npm run build`를 통해 빌드 성공을 확인하고, 에러 발생 시 로그를 분석하여 수정하는 자가 치유 루프를 실행한다.
* [ ] 3-3. 변경된 설계 사항이나 특이점은 `_spec.md` 하단 '변경 이력'에 실시간 기록한다.

---

### [Phase 4: 배포 및 피드백]

* [ ] 4-1. 모든 파일(코드+문서)을 포함하여 `git push`를 수행하고 Vercel 자동 배포 트리거를 발동한다.
* [ ] 4-2. 배포 완료 후 웹 UI에 해당 프로젝트 전용 'Live Preview' 링크와 '자가 검토 리포트'를 띄운다.
