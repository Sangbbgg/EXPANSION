Vercel에서 **404 에러**가 발생하는 것은 대개 기본 페이지(`app/page.tsx`)가 없거나, 배포 설정 시 빌드 결과물의 경로가 맞지 않을 때 발생합니다. 특히 Next.js 14 App Router를 사용하면서 스켈레톤 코드가 제대로 구성되지 않았을 가능성이 큽니다.

이 문제를 해결하고, 실제 기능 구현으로 넘어가기 위한 **업데이트된 작업 목록**을 드립니다. **0번 규칙(체크 및 자동 푸시)**을 Gemini CLI에게 인지시킨 후 하나씩 진행하세요.

* [x] 0-2. `next.config.mjs 파일 설정을 점검하고, Vercel 배포 시 App Router가 정상 인식되도록 수정해줘.`
* [x] 0-3. `배포된 웹 사이트에서 Supabase 및 Gemini API 환경 변수가 정상적으로 읽히는지 확인하는 'System Check' API route를 만들어줘.`
