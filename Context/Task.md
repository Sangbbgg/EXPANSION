### 해당 항목들을 구현할때 완료된 항목은 체크표시를 해줘

### [Phase 1: 기반 인프라 및 환경 구축]

1. `EXPANSION 프로젝트를 위한 Next.js 14(App Router) 기본 스켈레톤 코드를 생성하고 배포 환경을 설정해줘.`
2. `Gemini API 연동을 위한 백엔드 Route Handler를 작성하고 환경 변수(GEMINI_API_KEY) 연결을 확인해줘.`
3. `Tailwind CSS를 활용하여 대시보드의 기본 레이아웃(사이드바, 메인 컨텐츠 영역)을 구현해줘.`

### [Phase 2: 핵심 인터페이스 구현]

4. `관리자가 지시를 내릴 수 있는 실시간 채팅 입력창과 메시지 로그 UI를 만들어줘.`
5. `클라우드 터미널의 출력을 시뮬레이션하거나 중계할 수 있는 '라이브 로그 뷰어' 컴포넌트를 구현해줘.`
6. `프로젝트의 상태(기획/개발/테스트)를 한눈에 볼 수 있는 칸반 보드 또는 스테이지 인디케이터를 제작해줘.`

### [Phase 3: AI 워크플로우 엔진 자동화]

7. `입력된 프롬프트를 분석하여 파일 생성, 패키지 설치 등 CLI 명령어로 분리하는 'Task 분해 로직'을 작성해줘.`
8. `코드 작성 후 자동으로 npm run build를 실행하여 에러를 체크하는 '자가 검증 스크립트'를 생성해줘.`
9. `빌드 에러 발생 시 에러 메시지를 Gemini에게 다시 보내 수정을 요청하는 'Self-Healing' 루프를 구현해줘.`

### [Phase 4: 다중 프로젝트 및 데이터 관리]

10. `여러 개의 프로젝트를 생성하고 관리할 수 있도록 **Supabase(PostgreSQL)** 연동을 구현하고, 프로젝트 리스트를 클라우드 DB에 저장/불러오기 하는 기능을 만들어줘.`
11. `각 프로젝트별로 독립적인 채팅 내역과 작업 로그가 저장되도록 컨텍스트 분리 기능을 추가해줘.`
12. `완료된 프로젝트의 결과물을 미리 볼 수 있는 'Live Preview' 연결 버튼과 팝업 UI를 구현해줘.`

### [Phase 5: 최종 최적화 및 안정화]

13. `전체 UI의 반응형 디자인을 점검하고 관리자 승인이 필요한 'Checkpoint' 팝업 시스템을 완성해줘.`
14. `현재까지 구현된 모든 기능의 예외 처리를 수행하고, 최종 배포를 위한 최적화(Production Build)를 진행해줘.`
