# 📂 Task_AItoCLI.md (자동화 및 프로젝트 독립 관리 가이드)

본 파일은 프로젝트 **EXPANSION**의 자동화 루프 및 프로젝트 간 독립성, 그리고 자가 검토(Self-Audit) 프로세스를 정의합니다. Gemini CLI는 이 가이드를 준수하여 작업을 수행해야 합니다.

---

### [Phase 0: 기본 규칙 및 독립/추적 원칙]
- [x] 0-1. **[자동화 규칙]** Gemini CLI는 본 파일을 상시 모니터링하며, 작업 완료 시 즉시 체크([x]) 및 `git push`를 수행한다.
- [ ] 0-2. **[격리 원칙]** 모든 신규 프로젝트는 `/projects/[project_name]` 하위에 독립된 폴더로 생성한다. 기존 프로젝트나 시스템 코드를 직접 참조하지 않는다.
- [ ] 0-3. **[추적 강제]** 프로젝트 생성 시 반드시 `/projects/[project_name]/Context` 폴더를 만들고, 그 안에 단계별/기능별 체크리스트인 `[project_name]_spec.md`를 생성한다.
- [ ] 0-4. **[자가 검토]** 기능 구현이 완료될 때마다 AI는 코드를 재검토하고 변경 사항을 추적하여 `[project_name]_spec.md`를 업데이트(체크)해야 한다.
- [ ] 0-5. **[Git 프로세스 엄수]** 파일 생성 및 수정 후에는 반드시 `git add .` -> `git commit` 단계를 완료한 뒤 `git push`를 수행한다. 'Changes' 상태의 파일이 남아있는 채로 푸시를 시도하지 않는다.

### [트리거 및 강제 행동 지침]
- [ ] T-1. **[Pre-Action]** 모든 명령 수신 시, 관련 `.md` 파일을 읽어 현재 진행 상황을 먼저 파악한다. (생략 불가)
- [ ] T-2. **[Post-Action]** 파일 수정이 발생하면 10초 이내에 해당 프로젝트의 `Context` 문서를 수정 사항에 맞춰 동기화한다.
- [ ] T-3. **[Git-Trigger]** 모든 `git push` 전에는 반드시 `git status`를 실행하여 `.md` 파일이 Staging Area(`add` 상태)에 포함되었는지 육안으로 재확인한다.

### [Phase 1: 지시 수신 및 환경 구축]
- [ ] 1-1. 웹 UI 지시사항 수신 시 `/projects/[project_name]` 디렉토리를 생성하고 프로젝트 환경을 격리하여 세팅한다.
- [ ] 1-2. 해당 프로젝트 전용 `Context/` 폴더를 생성하고 요구사항을 분석하여 `[project_name]_spec.md` 초안을 작성한다.

### [Phase 2: 프로젝트별 독립 구현 및 문서 업데이트]
- [ ] 2-1. 정의된 `_spec.md`의 순서에 따라 기능을 구현하며, 타 프로젝트에 영향을 주지 않는지 매번 검증한다.
- [ ] 2-2. 개별 기능 구현 완료 시마다 AI가 직접 `/projects/[project_name]/Context/` 내 문서를 열어 진행률을 체크표시(`[x]`)로 변경 저장한다.
- [ ] 2-3. 구현된 기능 중 설계와 변경된 사항이 있다면 `_spec.md` 하단에 '변경 이력'을 추가하여 기록한다.

### [Phase 3: 빌드 검증 및 자가 치유]
- [ ] 3-1. 해당 프로젝트 폴더 내에서 `npm run build`를 실행하여 독립적인 빌드 성공 여부를 확인한다.
- [ ] 3-2. 빌드 실패 시 에러 로그를 분석하여 수정하고, 수정 과정을 `Context` 내 문서에 간략히 기록한 뒤 재검증한다.

### [Phase 4: 통합 Push 및 웹 배포 확인]
- [ ] 4-1. 구현된 코드와 업데이트된 `Context/` 폴더 내 마크다운 문서를 함께 포함하여 `git push`를 수행한다.
- [ ] 4-2. Vercel 배포 완료 후 웹 UI 상에 해당 프로젝트의 'Live Preview' 링크와 '기능 구현 완료 리포트(MD 기반)'를 노출한다.


---

## 🛠️ 관리자(User) 확인 사항
1. 신규 프로젝트 착수 시 AI가 `/projects` 하위에 전용 폴더를 만드는지 확인하십시오.
2. AI가 지시를 수행한 후 해당 프로젝트 폴더 내부의 `Context/` 파일이 실제 구현 내용과 일치하게 업데이트되었는지 점검하십시오.
3. 모든 작업 로그는 `EXPANSION` 메인 대시보드에서 실시간으로 확인할 수 있습니다.
